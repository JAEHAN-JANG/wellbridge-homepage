# 🏢 WellBridge 기업 홈페이지 & 관리자 시스템 (CMS)
**React (Vite) · Spring Boot · React-Admin · JWT · MySQL**

> 실무 환경에서 **기업 홈페이지 + 관리자(CMS)** 를 단독으로 설계/구현하고, 내부 운영 환경에 배포까지 수행한 프로젝트입니다.  
> 회사 정책상 소스코드는 비공개이며, 본 문서는 **구조/기능/기술적 의사결정 중심의 케이스 스터디**로 정리했습니다.

---

## 1. 프로젝트 개요

- **프로젝트명**: WellBridge 기업 홈페이지 및 관리자 시스템
- **개발 기간**: 2024.10 ~ 2024.12
- **수행 방식**: **단독 개발**
- **배포 현황**: 내부 운영 환경 배포(실사용 목적)
- **코드 공개 여부**: 비공개 (사내 자산 / 보안 정책)
- **키워드**: CMS 구축, REST API 설계, JWT 인증/인가, 파일 업로드/서빙, 운영 관점 설계

---

## 2. 담당 역할 (기여도 100%)

### ✅ 담당 범위
1) **기업 홈페이지 프론트엔드 전체 구현 (React + Vite + Tailwind)**  
2) **관리자 페이지(CMS) 설계 및 구현 (React-Admin 커스터마이징)**  
3) **Spring Boot 기반 REST API 설계 및 연동**  
4) **JWT 기반 관리자 인증/인가 체계 구현**  
5) **이미지 업로드 및 파일 관리 구조 설계 (Multipart + 서버 저장 + URL 규칙 통일)**  
6) **Nginx 기반 운영 환경 구성/디버깅 (프론트 정적 서빙 + API 프록시)**  

### ✅ 내가 만든 “운영 가능한 포인트”
- 관리자 관점에서 **콘텐츠 등록/수정 흐름을 최소 클릭으로 단순화**
- 공개 API(`/api/public`)와 관리자 API(`/api/admin`)를 분리해 **운영/보안 경계 명확화**
- 파일 업로드/서빙 규칙을 통일해 **리소스가 늘어나도 유지보수 가능한 구조**로 설계

| 영역 | 사용 기술 |
|---|---|
| Frontend | React (Vite), Tailwind CSS |

## 4. 아키텍처 & API 설계

### 4.1 시스템 구성
- **Corporate Homepage**: 사용자 공개 페이지 (정적 리소스 + Public API 연동)
- **Admin CMS**: 콘텐츠 관리용 (관리자 인증 필요)
- **Backend**: Spring Boot REST API
- **DB/File Storage**: MySQL + 서버 파일 저장소

### 4.2 API 경로 분리 전략
- `/api/public/**` : 홈페이지에서 사용하는 공개 API (Read 중심)
- `/api/admin/**` : 관리자 기능 API (JWT 인증 필요)

> 운영 중 권한/보안 이슈를 줄이기 위해 “경로 레벨”에서 관리자/공개 영역을 물리적으로 분리했습니다.

## 5. 주요 기능

### 🏠 기업 홈페이지
- 메인/회사소개/사업영역/조직도/지점안내/커뮤니티/문의 페이지
- Tailwind 기반 **반응형 UI(PC/Tablet/Mobile)**

### 🧩 관리자 시스템(CMS)
- 뉴스/게시글/FAQ/연혁/정책/조직도 이미지/지점/문의/운영사진 등 콘텐츠 관리
- CRUD + 이미지 업로드 + 노출/정렬 관리
- 관리자 로그인(JWT) 기반 접근 제어

### 🖼 이미지 업로드/서빙
- 관리자에서 이미지 업로드 → 서버 저장 → DB에 메타데이터 저장 → URL 규칙으로 조회
- 리소스(뉴스/게시글/운영사진 등)가 늘어나도 동일한 방식으로 확장 가능하도록 공통화

## 6. 기술적 도전 & 문제 해결

### 6.1 이미지 처리 방식이 리소스마다 달라 유지보수가 어려움
**문제**
- 뉴스/게시글/운영사진 등 리소스가 늘면서 이미지 업로드/응답 규칙이 달라져 프론트 처리 복잡도 증가

**해결**
- 업로드/서빙 흐름을 공통화:  
  - 서버: 업로드 엔드포인트/저장 규칙/응답 포맷 통일  
  - 클라이언트: React-Admin `dataProvider`에서 이미지 URL 처리 로직 분리/일원화  
- 결과적으로 “새 리소스 추가” 시 이미지 처리 코드 중복이 크게 줄어듦

---

### 6.2 관리자 인증/인가 흐름을 CMS에 안정적으로 붙이기
**문제**
- React-Admin은 기본적으로 CRUD 프레임워크라 인증/헤더/에러처리를 프로젝트 정책에 맞게 커스터마이징해야 함

**해결**
- JWT 로그인 → 토큰 저장 → `dataProvider`에서 모든 요청에 `Authorization: Bearer <token>` 주입
- 인증 실패(401/403) 시 관리자 화면에서 명확한 UX 처리(리다이렉트/세션 만료 안내)

---

### 6.3 뉴스 등록 시 “저작권 위험 + 입력 번거로움”
**문제**
- 외부 기사 기반 뉴스 등록 시 제목/요약/대표이미지 수기 입력 → 누락/오타 발생
- 원문 링크 누락 시 **출처 미표기**로 저작권 리스크

**해결**
- **OG(Open Graph) 태그 자동 수집 기능 구현**
  - 관리자: URL만 입력
  - 서버: OG 태그 파싱 → 제목/요약/대표이미지 자동 추출 → 원문 링크 저장 강제
- 결과
  - 뉴스 등록 속도 향상
  - 출처 구조 강제 → 저작권 리스크 사전 예방
  - 운영자 입장에서 “실제로 쓰는 기능”으로 평가
